# 光パズル — WDM / 波長割当 (Increment 0 / MVP)

光伝送の **WDM(波長多重)/ RWA(経路と波長の割当)** を抽象パズル化したブラウザゲームです。
1本のファイバ(管)には色違いの信号を同居させられますが、同じ色は同居できません。
全ての「依頼」を衝突なく繋げばクリアです。Flow Free 系の「占有・非共有」パズルと逆に、
**多重(共有)が遊びの主役** であることが特徴です。

> 本リポジトリはスコープ定義書セクション3(MVP / Inc.0)の実装範囲です。
> 信号品質予算(距離・損失・OSNR)、非線形ペナルティ、速度ティア、集約などは後続Incrementで追加します。

## 遊び方・操作

1. 画面下の **依頼**(例: `d1 A→C`)をタップして選択する。
2. 盤面の **ノードを始点から順にタップ** して経路を作る(ファイバで隣り合うノードのみ進める)。
   - `↩ ひとつ戻る` で1手戻す / `✕ 経路クリア` で始点だけに戻す。
   - 直前に通ったノードを再タップしても1手戻せます。
3. **パレット**(色=波長)から色をタップして、選択中の依頼に割り当てる。
4. 同じファイバを **同じ色が2本以上** 通ると、そのファイバが **赤く** 衝突表示されます。
   色を変えるか経路を変えて解消しましょう。
5. 全依頼が「経路接続済み + 色割当済み + 衝突なし」になると **クリア**。

- 各依頼の状態: `✓ OK` / `⚠ 衝突` / `…未完`。
- 任意目標として **par(目標色数)** を表示。クリアに必須ではありませんが、より少ない色で解くと `★par達成`。
- 進捗(経路・色・クリア状況)は **localStorage** に自動保存され、再訪時に復元されます。

## 同梱レベル(4面)

| # | id | 狙い |
|---|---|---|
| 1 | L0-01 多重 | 同じ管に色違いを同居させる(多重)の体感 |
| 2 | L0-02 再利用 | 経路を分ければ同じ色を再利用できる(空間再利用) |
| 3 | L0-03 分散 | 限られた色で複数依頼を2経路へ振り分ける |
| 4 | L0-04 割当 | 共有ファイバ上の色制約を解く(RWAの核) |

## 設計判断

- **1ファイル HTML + vanilla JS + SVG / 外部依存ゼロ**: ビルド不要でスマホブラウザで即動作。
  GitHub Pages に `index.html` を置くだけで公開できる。MVP規模では SVG が軽く扱いやすい
  (要素が増えて重くなれば Canvas へ移行可能)。
- **localStorage**: サーバ不要で進捗・クリア状況を保存。
- **モバイルファースト**: タッチ前提。ノードは当たり判定を広げ(半径22pxの透明円)、指で押しやすくしている。

## データモデルとレベル追加方法

レベルは `index.html` 内の `LEVELS` 配列に定義しています(セクション6スキーマ準拠)。
コードとは分離した純データなので、配列に1要素足すだけでレベルを追加できます。

```json
{
  "id": "L0-01",
  "name": "多重",
  "par": 2,
  "palette": ["#e6194B", "#4363d8"],
  "nodes":   [{ "id": "A", "x": 60, "y": 120 }, { "id": "B", "x": 160, "y": 120 }],
  "fibers":  [{ "id": "f1", "a": "A", "b": "B" }],
  "demands": [{ "id": "d1", "src": "A", "dst": "C" }]
}
```

- 座標は SVG の `viewBox="0 0 320 240"` 基準。
- `palette` の要素数が使える色数(波長数)の上限。
- プレイヤーの解は内部的に `{ demandId: { pathNodes:[...], color: idx|null } }` で保持。
  衝突判定は「各ファイバを通る依頼の色集合に重複があれば衝突」。

> 補足: スコープ定義書ではレベルJSONを別ファイルに分離する想定ですが、
> 「外部依存ゼロ」かつ `file://` でスマホ単体動作させるため、本MVPでは `LEVELS` として埋め込んでいます
> (`fetch` は `file://` でCORS制約に当たるため)。データ構造は分離しているので別ファイル化への移行は容易です。

## 公開手順(GitHub Pages)

1. このフォルダを GitHub リポジトリに push(`index.html` がルートにあること)。
2. リポジトリの **Settings → Pages** で、Source を `main` ブランチ / `/ (root)` に設定。
3. 数十秒後に発行される URL を開く(`https://<user>.github.io/<repo>/`)。
4. スマホブラウザでアクセスして動作確認。

ローカル確認は `index.html` をブラウザで直接開くだけでも動作します(サーバ不要)。
