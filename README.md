# いろつなぎ — 点をつなぐ色パズル

点と点を線でつなぐパズルです。**同じ管(くだ)でも、色がちがえば一緒に通れる**のが特徴で、すべての線をぶつからずにつなげたらクリア。スマホブラウザでそのまま遊べます(インストール不要)。

▶ **公開URL: https://dabidabide.github.io/optical-puzzle/**

---

## あそびかた

1. **つなぐ線をえらぶ** … 下のリストから線をタップ。青い点が出発、紫の点がゴール。
2. **点を順にタップして道をつくる** … となりの点(管でつながった先)へ進めます。「ひとつ戻る」でやり直し可。
3. **色をえらぶ** … 上の色をタップして、その線の色を決めます。

ルールはひとつだけ:

> **同じ管を「同じ色」が2本通るとぶつかります(赤く表示)。**
> 色を変えるか、道を変えて直しましょう。色さえ違えば、1本の管に何本でも一緒に通せます。

ぜんぶの線をぶつからずにつなげたら **クリア**。さらに少ない色数でクリアすると **ボーナス(✦)** がつきます(やりこみ向けの任意目標)。

## 上級モード ✦(任意)

画面上部の「上級」タブで、深い遊びの5面に切り替えられます(ふつうの8面はそのまま)。基本ルールに加えて、用語を出さずに4つの仕掛けが入ります。

| 名前 | 加わる仕掛け | 中身(プレーン言葉) |
|---|---|---|
| とどく力 | 距離の予算 | 線に「とどく力」。道が長い/混むと消耗し、足りないと届かない |
| こみ道 | 混雑 | 同じ管を多くの線が通るほど、その管の信号が弱る(詰めすぎ注意) |
| いろがえ | 色がえ中継点 | オレンジの◇を通ると色を途中で変えられる(区間ごとに色を選択) |
| はやさ | 速さ等級 | 速い線(»の数)ほど「とどく力」を多く使う=短い道が必要 |
| そうごう | 上記の総合 | 距離・混雑・速さを同時に成立させる |
| おおぞら | 大型・総合 | 広い回廊網で2+2に分けて混雑を抑えつつ速い線を通す |
| いろのちから | 色ごとに距離差 | 色(波長)ごとに「強さ」が違う(×1.0/×1.3/×1.6)。弱い色ほど距離コスト増→遠い線は強い色で(仕様書: 波長分散/帯) |
| リレー | 中継で回復 | 緑の◇を通ると「とどく力」が回復。区間ごとに予算判定で、遠くへは中継を挟んで届かせる(仕様書: リジェネ/3R) |
| まもり | 予備の道(保護) | 各線に「本線」＋「予備(破線)」の2本。予備は本線と管を共有しない別経路に。両方つながってOK(仕様書: 1+1保護/SNCP) |
| にじゅう | 保護＋混雑＋予算 | 六角リング網で3本を本線/予備の二重化。混雑と予算をさばく達人面 |

**見える化**: 各管に「長さ(コスト)」を数字表示。線ごとに「とどく力」ゲージと内訳(**道のり ＋ こみ ×はやさ**)を表示し、引く前に消費を見積もれます。混雑している管は**オレンジで太く「×本数」**表示。速さは「»の数 ×倍率」で明示。

> これらは仕様書の Inc.1(信号品質予算・非線形・波長変換)と L1(速さティア)を、専門用語を出さずに抽象化したもの。コスト式 `(道のり ＋ こみ) ×はやさ`(混雑=共有本数に比例)は `_adv_solver.py` でゲーム側(`index.html`)と一致させ、全5面が設定予算で解けることを検証済みです(`py _verify.py`)。仕様書の方針「まず軽く、深さは選択制」に沿い、上級は**任意のモード**として分離しています。

## 設計判断

- **1ファイル HTML + vanilla JS + SVG / 外部依存ゼロ** … ビルド不要でスマホブラウザで即動作。GitHub Pages に `index.html` を置くだけで公開できる。
- **localStorage** … サーバー不要で進捗を保存。
- **モバイルファースト** … タッチ前提。点は当たり判定を広げ(半径23pxの透明円)、指で押しやすくしている。
- **言葉はやさしく** … 画面上は「点・管・線・色」など日常語のみ。専門用語は出していません(背景は下記)。

## データモデルとステージ追加

ステージは `index.html` 内の `LEVELS` 配列に純データとして定義(コードと分離)。配列に1要素足すだけで追加できます。

```js
{
  id: "v1",
  name: "はじめて",
  palette: ["#e6194B", "#4363d8"], // 使える色(=色数の上限)
  par: 2,                           // ボーナスがつく目標色数
  nodes:   [{ id:"A", x:60, y:120 }, ...],   // 点(座標はviewBox 320x240基準)
  fibers:  [{ id:"f1", a:"A", b:"B" }, ...], // 管(点と点をつなぐ)
  demands: [{ id:"d1", src:"A", dst:"C" }]   // つなぐ線(出発→ゴール)
}
```

- プレイヤーの解は内部的に `{ demandId: { pathNodes:[...], color: idx|null } }` で保持。
- 衝突判定は「各管を通る線の色集合に重複があれば衝突」。
- ステージを足したら、開発用の `_design_solver.py`(Python)で可解性と最少色数を検証できます:
  ```
  py _design_solver.py
  ```

## 背景・モチーフ(技術的な元ネタ)

このゲームは光通信の **WDM(波長分割多重)/ RWA(経路と波長の割当)** という考え方を、用語を出さずに抽象パズル化したものです。

- 「管」= 光ファイバ、「色」= 波長(光の色)、「線」= つなぎたい通信。
- 1本のファイバに複数の波長(色)を同時に流せる(=多重)が、同じ波長は同居できない、という実際の制約がそのままルールになっています。
- Flow Free 系の「線が交差・共有できない」パズルとは逆に、**共有(多重)が遊びの主役**である点が特徴です。

将来的な拡張(任意・選択制)の構想:

| 段階 | 追加要素 |
|---|---|
| Inc.1 | 信号品質の予算(距離・損失)、混雑ペナルティ、色を変換できる中継点 |
| Inc.2 | 速度ティアと到達距離、電気区間のボトルネック |
| Inc.3 | 束ねる(集約)、ループの遮断 |

> これらは「クリアには必須でない、上級者向けの任意モード」として段階的に足せるよう、`CONFIG` フラグで切り替えられる設計にしています。本線は常にやさしいまま保ちます。
